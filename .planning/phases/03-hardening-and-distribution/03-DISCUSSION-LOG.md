## Distribution channels

| Option | Description | Selected |
|--------|-------------|----------|
| Brew formula via custom tap + signed/notarized .pkg | Both deliver DIST-02's stable absolute path. Brew @ /opt/homebrew/bin or /usr/local/bin (per arch); .pkg @ /usr/local/bin. uvx stays as dev path. | ✓ |
| .pkg only | Skip brew complexity; smaller maintenance surface. | |
| Brew only | Skip Apple Developer cert hassle; brew works for most. | |
| uvx only (status quo) | Cheapest; ships TCC churn problem documented but unsolved. | |

**Auto choice:** Both brew + .pkg.
**Rationale:** DIST-02 mandates "stable absolute path" — both achieve it. End users want easy install; brew suits CLI-savvy, .pkg suits non-technical. uvx remains dev path with documented caveat.

## .pkg structure

| Option | Description | Selected |
|--------|-------------|----------|
| Self-contained Python venv bundle | Relocatable venv with pyobjc + mcp pre-installed; launcher shell script invokes bundled interpreter. No host Python required. | ✓ |
| Shell out to `system pip install` at install time | Smaller .pkg but requires host Python 3.12 + network at install time. | |
| Compile to single binary via PyInstaller | Big binary; pyobjc dynamic-lib handling fragile. | |

**Auto choice:** Self-contained venv bundle.
**Rationale:** Best end-user experience (no Python prereq); largest .pkg (~50MB) but acceptable for desktop install.

## Code signing pipeline

| Option | Description | Selected |
|--------|-------------|----------|
| Extend release.yml with pkg-build job; Apple cert from secrets; productsign + notarytool + stapler; if-secret-exists guard | Automates the manual Apple step every release; community forks still get unsigned builds. | ✓ |
| Manual signing per release | Maintainer signs locally; brittle, error-prone. | |
| Skip signing entirely | macOS Gatekeeper blocks unsigned .pkgs by default; hostile UX. | |

**Auto choice:** Automated pipeline.
**Rationale:** Manual signing is a per-release foot-gun; cert-secret guard keeps community contributors unblocked.

## FTS5 shadow index

| Option | Description | Selected |
|--------|-------------|----------|
| Sidecar SQLite at ~/Library/Application Support/whatsapp-desktop-mcp/fts.sqlite; lazy build on first search; incremental refresh on Z_VERSION match | Separate file; simple lifecycle; doesn't bloat rate-limit.db. | ✓ |
| In-memory FTS rebuilt per session | Cold-start latency every server start. | |
| Add FTS table to existing rate-limit.db | Co-location confuses lifecycles; rate-limit lives forever, FTS rebuilds on schema change. | |
| FSEvents watcher continuously updating | Background daemon; out of scope for stdio MCP. | |

**Auto choice:** Sidecar with lazy/incremental build.
**Rationale:** Simplest architecture; no daemon; matches Phase 2 sidecar pattern.

---

## FTS5 schema choice

| Option | Description | Selected |
|--------|-------------|----------|
| FTS5 unicode61 with diacritic-removal | Matches naïve user expectations (café == cafe); standard tokenizer. | ✓ |
| FTS5 trigram | Substring matching; heavier index; noisier ranking. | |
| FTS5 porter stem | English-only; degrades for multilingual chats. | |

**Auto choice:** unicode61 + diacritic-removal.
**Rationale:** Matches what real users want; lighter than trigram; multilingual-friendly.

## Doctor schema-version range

| Option | Description | Selected |
|--------|-------------|----------|
| Add supported_version_range tuple + degraded_mode_warning string to SchemaFingerprint | Doctor structurally surfaces "WA version untested" without crashing read tools. | ✓ |
