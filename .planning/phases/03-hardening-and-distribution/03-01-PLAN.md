Ship the FTS5 shadow index that upgrades `search_messages` from the Phase 1 LIKE scan to a ranked sub-second FTS5 query against a sidecar SQLite at `~/Library/Application Support/whatsapp-desktop-mcp/fts.sqlite`. Output: New `reader/search_fts5.py` module + dispatcher refactor of `tools/search_messages.py` + `server.fts5_mode` module attr + `--fts5-mode` CLI arg + Wave-0 unit tests for both modules.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User search query → FTS5 MATCH operator | Untrusted UTF-8 string crosses into FTS5's MATCH mini-language — operator characters (`*`, `"`, `(`, `)`, `:`, `-`, `+`, `^`) would otherwise parse as syntax. |
| FTS sidecar file → on-disk persistence | Persistent state at `~/Library/Application Support/whatsapp-desktop-mcp/fts.sqlite` outlives the process; another local process with write access to the user's home dir could tamper. |
| Reader → ChatStorage RO joinback | This boundary is unchanged from Phase 1; we re-affirm REL-05 D-24 (the only sender→reader-connection edge is `sender/verify.py`, and Phase 3 adds zero new sender→reader edges). |
