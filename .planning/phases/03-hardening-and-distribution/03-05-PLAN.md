## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Maintainer's `~/Library/` production state → smoke run sandbox | The autouse fixture's monkey-patches must completely isolate test state from production paths (Pitfall 5). |
| RUN_LIVE_WHATSAPP env var → smoke gate | A misconfigured env in CI could accidentally fire the smoke against runners without WhatsApp.app installed (would hang or false-fail). |
| `git tag v*` push → release.yml pipeline | The tag push triggers an irreversible PyPI publish (PyPI does NOT allow re-uploading a version after deletion). The dry-run discipline (v0.0.1-rc1) prevents accidental v0.1.0 burn. |
| Real WhatsApp send during Phase 2 smoke | The smoke fires actual UI sends to test chats per Phase 2 D-23 (RUN_LIVE_BURN_BUDGET gating) — counts against WhatsApp ToS thresholds (Phase 2 P14 mitigation). |
