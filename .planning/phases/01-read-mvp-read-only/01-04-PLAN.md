## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → tool input | Pydantic-typed parameters (int, str, bool, optional Unix-second timestamps) validated by FastMCP's JSON-schema introspection before the tool body runs. |
| Tool body → reader | Tool calls `await reader.<fn>(...)` — the reader is the only code path with sqlite access. Tools never construct SQL. |
| Tool response → MCP client | The 60k-char budget + opaque cursor + truncation flag bound how much data crosses the boundary in a single response (P9 mitigation). |
| Cursor (opaque) → tool input | LLM may fabricate or tamper with cursors; `decode_cursor` raises `CursorError`; the tool then surfaces a structured `ValueError` and the LLM restarts pagination. |
| `chat_id` argument → reader filter | An LLM may pass a `chat_id` it never received from `list_chats` / `search_contacts`; the reader returns an empty result (no chat with that PK) — no privilege escalation because the LLM already has FDA-granted read access to the user's DB. |
