PermissionState = Literal["granted", "denied", "whatsapp_not_installed"]

Note: Python's Phase-1 `DoctorReport` mandates `db_path: str` (NOT Optional) and `schema_fingerprint: SchemaFingerprint` (NOT Optional) and `coverage_summary: Coverage` (NOT Optional). CONTEXT D-13 explicitly says Rust v0 returns `None` for `schema_fingerprint`, `last_message_ts`, `coverage_summary`. The cross-binary parity test (Plan 04-03) compares only the v0-populated subset (`{full_disk_access, automation_whatsapp, accessibility, db_path, whatsapp_app_version}`) to handle this gap. Rust serde models use `Option<>` for the deferred fields with `#[serde(skip_serializing_if = "Option::is_none")]` so they appear as JSON `null`/absent and don't violate Python's deserialization (which would fail if Rust emitted them).
