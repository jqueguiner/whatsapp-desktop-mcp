# Hard rules carried from CLAUDE.md
- Stdout is the JSON-RPC channel; logging goes to stderr. Single `print()` corrupts the protocol. P-PHASE0-01 is the gating pitfall.
- Stdio only. `mcp.run()` defaults to `transport='stdio'`; do not pass any `transport=` argument that could be a different value.
- Reader and Sender must not import each other (this plan touches neither; Reader/Sender stay empty).

| Boundary | Description |
|----------|-------------|

| process ↔ macOS TCC | Process inherits TCC grants of its requesting binary (`sys.executable` under `uvx`) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-04 | Tampering | Stray stdout output (a `print`, a logger misconfigured to stdout, a noisy third-party import) corrupts JSON-RPC framing, silently breaking the Claude Desktop connection | mitigate | `logging.basicConfig(stream=sys.stderr, ...)` is the FIRST executable statement in `server.py` (D-05); ruff T201 lint-blocks `print` from Plan 01's pyproject.toml; Plan 04's `test_stdout_purity.py` is the CI gate (P-PHASE0-01) |
| T-00-05 | Tampering | A future maintainer drops to lower-level `mcp.server.Server` or imports the wrong FastMCP distribution (`fastmcp` vs `mcp.server.fastmcp`), introducing a different transport or different annotation shape | mitigate | Plan 02's `<verify>` greps for both forbidden patterns (`from fastmcp ` and `transport=`); D-03 locks FastMCP-only and Anthropic-SDK-only |
| T-00-06 | Elevation of privilege | An HTTP listener is introduced (the `lharries/whatsapp-mcp` CVE class) | mitigate | `mcp.run()` is the only call; no `transport=` argument anywhere; CLAUDE.md §5 (no HTTP/TCP/UDP listener); D-04 |
| T-00-07 | Information disclosure | The CLI surfaces a Python traceback to stdout instead of stderr, corrupting JSON-RPC framing AND leaking install-path internals | mitigate | argparse writes to stderr by default; uncaught exceptions in `main()` propagate to the runtime which writes the traceback to stderr (Python default); the `--version` path exits via `argparse → SystemExit` before any potentially-throwing import runs |
