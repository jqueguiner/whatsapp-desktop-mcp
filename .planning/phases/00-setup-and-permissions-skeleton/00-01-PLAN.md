# Phase Goal — User Story
**As a** Claude Desktop user, **I want to** add one line to `claude_desktop_config.json`, restart Claude Desktop, and call `doctor`, **so that** I get a structured, actionable macOS-permissions report end-to-end from my MCP client.

# Decisions implemented by this plan
- D-01 (src-layout `whatsapp_desktop_mcp`)
- D-02 (hatchling build backend, console script `whatsapp-desktop-mcp = whatsapp_desktop_mcp.cli:main`)
- D-13 (ruff `>=0.6`, T201 + E + F + I + B + UP + TID, line-length 100)
- D-14 (mypy `>=1.10`, `--strict --warn-unreachable`, files = src + tests)
- D-15 (pytest `>=8.2` + `pytest-asyncio` + `pytest-subprocess>=1.5`, `tests/unit` and `tests/integration` layout, `markers` declared)
- D-19 (`requires-python = ">=3.12"`)
- REL-05 (Reader/Sender packages exist as siblings from day one — enforced by directory structure, not convention)

# Hard rules carried from CLAUDE.md
- Reader and Sender packages MUST NOT import each other (Phase 0 satisfies this by leaving both empty).
- Never write to `ChatStorage.sqlite` (this plan ships no SQLite-touching code).
- No HTTP / TCP / UDP listener (this plan ships no transport code).
