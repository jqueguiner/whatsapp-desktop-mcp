# `pytest-subprocess` quick reference (verified API surface, RESEARCH.md §"Code Examples / Automation-probe mocking")
- `fp` is the fixture name; `pytest-subprocess>=1.5` provides it.
- `fp.register([cmd, arg1, arg2, ...], stdout=b"...", stderr=b"...", returncode=N)` queues a faked invocation.
- The test calls the production code; production's `asyncio.create_subprocess_exec` resolves to the registered fake.
